# Code of Conduct

BurnPast uses a simple contribution standard: be respectful, stay on topic, and keep technical discussions constructive.

## Expected Behavior

- Treat other contributors with respect.
- Critique code, designs, and tradeoffs rather than people.
- Be clear when discussing security risk, uncertainty, or limitations.
- Keep vulnerability details private until a fix is available.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing another person's private information.
- Sharing real secrets, tokens, keys, or credentials in issues, pull requests, or discussions.
- Publicly posting exploit details for an unfixed vulnerability.

## Enforcement

Maintainers may edit, hide, or remove content and may block users who repeatedly ignore these rules.
